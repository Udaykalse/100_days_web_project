Selector Specificity: The Hidden Rulebook

M
Mario Rossi 2 hours ago · 5 min read

Every CSS rule you write enters a silent competition. The browser evaluates specificity scores, source order, and cascade layers to decide which rule wins. Understanding this mechanism transforms how you write CSS.

The specificity algorithm assigns a three-part score to every selector. Inline styles dominate with a score of (1,0,0,0), ID selectors contribute (0,1,0,0), class and attribute selectors add (0,0,1,0), and element selectors score (0,0,0,1). The browser reads these like coordinates — a single ID always outweighs any number of classes.

CSS 
@layer reset, theme, layout, content, components, utility;

/* Specificity: (0,0,1,0) — class selector */
.card { background: white; }

/* Specificity: (0,1,0,0) — ID selector, always wins */
#hero .card { background: teal; }

/* Layer order beats specificity! */
@layer utility {
  .card { background: coral; } /* Wins via layer priority */
}

Cascade layers introduced in CSS 2022 change the game entirely. A rule inside a higher-priority layer wins over a rule with higher specificity in a lower-priority layer. This unlocks clean, predictable CSS architecture at scale.

The order you declare layers matters enormously. Later-declared layers have higher priority. This means you can safely import third-party libraries into a low-priority layer without worrying about specificity wars.
